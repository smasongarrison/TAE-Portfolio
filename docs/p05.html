Portfolio 5: Interactive Spotify Track Explorer

Goal I wanted to try my luck and create a small interactive data tool using R Shiny. I am using a Spotify Tracks dataset from Kaggle. Instead of making one static graph, I wanted users to be able to choose a music genre and compare different Spotify audio features. My goal is to practice building something that feels more like a usable product or mini-dashboard, while still keeping the project simple enough to finish for my portfolio.

Here is the link to the Kaggle dataset page: https://www.kaggle.com/datasets/maharshipandya/-spotify-tracks-dataset?select=dataset.csv

Each row in the dataset represents one track. The dataset includes song information such as track name, artist, album, genre, and popularity. It also includes audio features such as danceability, energy, loudness, speechiness, acousticness, instrumentalness, liveness, valence, and tempo.

The main question I explored was:

How do Spotify track features, such as danceability, energy, valence, tempo, and popularity, differ across genres?

Product

The final product is an interactive R Shiny explorer. A user can:

  1. Choose a genre.
  2. Choose an x-axis feature.
  3. Choose a y-axis feature.
  4. Filter tracks by popularity.
  5. View a plot that updates automatically.
  6. See the top songs in the selected genre.

Clean the data

I am going to start by selecting the columns that are most useful for the app. I want to keep the app simple and avoids overwhelming the user with too many variables.

spotify <- spotify_raw %>%
  select(
    track_name,
    artists,
    album_name,
    popularity,
    duration_ms,
    explicit,
    danceability,
    energy,
    loudness,
    speechiness,
    acousticness,
    instrumentalness,
    liveness,
    valence,
    tempo,
    track_genre
  ) %>%
  drop_na(track_name, artists, popularity, danceability, energy, valence, tempo, track_genre)

I will also convert the duration from milliseconds to minutes because minutes are easier to interpret.

spotify <- spotify %>%
  mutate(
    duration_min = duration_ms / 60000,
    explicit = if_else(explicit == TRUE, "Explicit", "Not explicit")
  )

Top Songs in Selected Genre

For this part, I had to check some suggestions from Claude to see how it works (cause I am new to this).

server <- function(input, output, session) {
  
  # This reactive object filters the dataset based on the user's choices.
  # Every time the user changes the genre or popularity range, this dataset updates.
  
  filtered_spotify <- reactive({
    spotify %>%
      filter(
        track_genre == input$genre,
        popularity >= input$popularity_range[1],
        popularity <= input$popularity_range[2]
      )
  })
  
  # This code creates the scatterplot.
  # The x-axis and y-axis change depending on what the user selects.
  
  output$spotify_plot <- renderPlot({
    ggplot(
      filtered_spotify(),
      aes(
        x = .data[[input$x_feature]],
        y = .data[[input$y_feature]],
        color = popularity
      )
    ) +
      geom_point(alpha = 0.6) +
      labs(
        title = paste("Spotify Tracks in the", input$genre, "Genre"),
        x = names(feature_choices)[feature_choices == input$x_feature],
        y = names(feature_choices)[feature_choices == input$y_feature],
        color = "Popularity"
      ) +
      theme_minimal()
  })
  
  # This code creates a table of the most popular songs in the selected genre.
  # I show only the most useful columns so the table stays readable.
  
  output$top_songs <- DT::renderDT({
    filtered_spotify() %>%
      arrange(desc(popularity)) %>%
      select(
        track_name,
        artists,
        album_name,
        popularity,
        danceability,
        energy,
        valence,
        tempo
      ) %>%
      head(10)
  })
}

Overview

Ok this was very fun to do!

This project helped me practice creating an interactive data product instead of only making a static graph. The app allows users to explore Spotify tracks by genre and compare different audio features. For example, users can compare danceability and energy, or look at whether more popular songs tend to have higher valence or tempo.

One thing I learned is that a Shiny app needs both data analysis and design thinking. I had to decide which variables were useful, which controls would be easy for users to understand, and how much information to show at one time. I ended up keping the app simple by using a few dropdowns, one plot, and one table.

I think as I learn more about this with more time, I would like to add more features such as comparing two genres, adding a search box for artists, or creating a recommendation tool based on the user’s preferred audio features.
